# Дополнительные данные для проекта AIProf

## Существующие данные в папке `data/`
- **aiTags.json**: Список AI-тегов для векторизации (code, name, ru, type, professions).
- **professions.json**: Профессии с ID, названием, колледжем и AI-профилем (вектором оценок).
- **questions.json**: Вопросы теста с кодом, текстом, типом, aiTags, весом, стадией, активностью.

## Дополнительные модели данных
На основе анализа данных и требований (пользователи - школьники 9-11 классов, планирующие поступить в колледж):

1. **AITag** (AI-тег):
   - Поля: code, name, ru, type (skill/cognitive/soft), professions
   - Коллекция: `aiTags`
   - Используется для категоризации вопросов и профессий.

2. **Обновленная User**:
   - Поля: id, name, school (школа), grade (класс 9-11), createdAt
   - Коллекция: `users`

## Возможные дополнительные данные/модели
- **College** (Колледж): id, name, description, professions (список доступных профессий). Если нужно отдельно от professions.json.
- **School** (Школа): id, name, region. Для аналитики по школам.
- **Analytics** (Аналитика): userId, metrics (процент завершения, время теста, etc.). Для сбора данных.
- **Feedback** (Обратная связь): userId, rating, comments. Для улучшения системы.

## Рекомендации
- Импортировать данные из JSON в Firestore при инициализации.
- Для aiTags использовать как справочник для сопоставления.
- Учитывать, что professions.json содержит aiProfile как объект, который можно векторизовать с aiTags.
